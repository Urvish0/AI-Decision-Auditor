AI Decision Auditor - Development Roadmap

Project Goal: Build a reasoning-first RAG system that audits decisions
by identifying risks, contradictions, missing information, and
confidence scores.

Overall Strategy: Core Engine → Agents → UI → Polish → Deploy

DAY 0 — Setup & Alignment - Create repo: ai-decision-auditor - Setup
backend (FastAPI) and frontend (Next.js) - Install PageIndex, LangGraph,
LLM SDK Done when: - Backend runs on localhost:8000 - Frontend runs on
localhost:3000

DAY 1 — PageIndex Core - Parse PDF → text - Build PageIndex tree - Store
as JSON Done when: - Tree prints correctly - You understand traversal

DAY 2 — Basic Query System - Input query - Traverse tree using LLM -
Extract nodes and generate answer Done when: - Works for 3–5 queries

DAY 3 — Planner + Critic - Planner: break query into steps - Critic:
find risks, missing info, failures Done when: - Structured output
(Answer, Risks, Missing Info)

DAY 4 — Multi-Agent Pipeline - Implement LangGraph workflow - Planner →
Navigator → Critic → Synthesizer Done when: - Agents run sequentially
and improve output

DAY 5 — Verifier Agent - Detect contradictions across sections Done
when: - At least one conflict detected

DAY 6 — Frontend - Chat UI - File upload - Display response + reasoning
trace Done when: - Demo-ready

DAY 7 — Polish - Error handling - Logging - Prompt tuning - Confidence
score Done when: - Feels like a product

DAY 8–10 — Deployment - Deploy backend (Railway/Render) - Deploy
frontend (Vercel) - Add demo video - Write README

Rules: 1. Don’t skip days 2. Don’t over-engineer early 3. Ship imperfect
→ improve 4. Test with real docs
